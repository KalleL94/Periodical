# Worklog 2026-06-06

## Summary

Today focused on codebase audit follow-up, production hardening, regression fixes, and GitHub issue cleanup.

## Completed changes

### Production secret hardening

- Added production validation for `SECRET_KEY` so known placeholders and short values are rejected when `PRODUCTION=true`.
- Updated deployment compose configuration to require `SECRET_KEY`.
- Added tests for production secret validation.
- Added `scripts/configure_prod_secret.sh` to configure `/etc/periodical.env`, create a systemd override, reload systemd, and optionally restart the service.
- Verified the script was run on production and `ica-schedule.service` restarted successfully.

PR: #255 `security: harden production secret validation`

### Database URL configuration

- Updated database initialization to read `DATABASE_URL` from the environment.
- Kept the existing SQLite path as the default for local/dev use.
- Applied SQLite-only connection arguments and PRAGMA setup only when the active backend is SQLite.
- Added tests for default URL, environment override, and SQLite/non-SQLite connect args.

PR: #256 `fix: read database url from environment`

### Overtime upsert behavior

- Changed overtime creation so one user/date has one overtime shift row.
- Existing rows are updated instead of duplicated.
- Legacy duplicate rows for the same user/date are cleaned up during update.
- Added tests for create, update, separate dates, and duplicate cleanup.

PR: #257 `fix: upsert overtime shifts by day`

### Shift swap conflict handling

- Added conflict detection for affected user/date slots in shift swaps.
- New proposals are blocked when either participant already has a pending or accepted swap on an affected date.
- Accepting a pending swap is blocked if it conflicts with an already accepted swap.
- Added focused regression tests for pending conflict, accepted conflict, and allowed non-conflicting acceptance.

PR: #258 `fix: block conflicting shift swaps`
